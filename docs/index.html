Analyse your GitHub
PRs with AI

A hands-on POC that fetches pull request data from your GitHub repos, summarises it with pandas, sends it to an LLM, and delivers an AI-written engineering report straight to your inbox.

Python 3.9+ PyGithub pandas Claude API Streamlit smtplib
// what you build

Four scripts.
One complete pipeline.

Each script teaches you a distinct layer of a GenAI application — from data collection to AI reasoning to delivery.

01
01_collect_github_data.py
GitHub Data Collector
Connects to the GitHub API using PyGithub and fetches PR metadata across your repos — title, state, CI check results, review comments, time open, author. Saves everything to pr_data.csv.
REST APIs PyGithub pandas CSV GitHub PAT
02
02_analyse_with_ai.py
AI Analyser
Reads the CSV, aggregates it into per-repo and per-author stats using pandas groupby, then builds a structured prompt and sends it to Claude or OpenAI. Returns a human-readable engineering report. Supports FREE MODE — paste the prompt into claude.ai manually if you have Claude Pro.
Prompt Engineering LLM APIs Token Cost Data Compression FREE MODE
03
03_send_email.py
Email Delivery
Reads the AI report, wraps it in a styled HTML email template, and sends it via Gmail SMTP. Uses Python's built-in smtplib — no extra libraries needed. Schedule with cron to get weekly reports automatically.
SMTP Protocol HTML Email Gmail App Passwords cron jobs
04
04_dashboard.py
Streamlit Dashboard
A dark-themed web dashboard that visualises PR health metrics with Plotly charts — merge rates, CI failure rates, author patterns, PR volume over time — plus the full AI report rendered inline.
Streamlit Plotly Data Viz Caching
// genai concepts you learn

What this POC
actually teaches you.

Each script is a deliberate lesson in a real-world GenAI pattern used in production systems.

🪟
Context Windows
LLMs can only read a limited amount of text at once. Learn why we summarise 500 CSV rows into a compact prompt instead of sending raw data.
✍️
Prompt Engineering
The 4 ingredients of a great prompt: Role, Context, Task, Format. See exactly how prompt structure changes the quality of the AI's output.
🪙
Token Economics
APIs charge per token. Learn to estimate cost before running, read usage stats from responses, and calculate exact cost in INR per run.
🔌
API Patterns
The messages[] format is an industry standard. Once you know Claude's API, switching to OpenAI or Gemini is just changing 3 lines of code.
🗜️
Data-to-Prompt Compression
Raw data → pandas aggregation → structured text → LLM. This pipeline pattern appears in every serious GenAI data app.
🆓
FREE MODE Pattern
No API key? The script generates a ready-to-paste prompt for claude.ai. A practical pattern when prototyping before committing to API costs.
// quickstart

Run the full pipeline
in 4 commands.

  • Python 3.9 or higher installedpython --version
  • GitHub Personal Access Token (read:repo scope)github.com → Settings → Developer Settings
  • Claude Pro account or Anthropic API keyconsole.anthropic.com
  • Gmail App Password for email deliverymyaccount.google.com → Security
# 1. Install dependencies
pip install PyGithub pandas anthropic streamlit plotly

# 2. Collect PR data from your GitHub repos
python 01_collect_github_data.py

# 3. Analyse with AI (FREE_MODE=True uses claude.ai)
python 02_analyse_with_ai.py

# 4. Email the report to your inbox
python 03_send_email.py

# 5. Launch the visual dashboard
streamlit run 04_dashboard.py
# Optional: run everything weekly with cron (every Monday 9am)
crontab -e
0 9 * * 1 cd ~/GithubAIPOC && python 01_collect_github_data.py && python 02_analyse_with_ai.py && python 03_send_email.py
